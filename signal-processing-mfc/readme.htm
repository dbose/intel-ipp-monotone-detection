Overview

The Using the Intel® IPP Signal Processing Functions with the MFC sample demonstrates how to use IPP Signal Processing functions in MFC application.

The sample is MFC application with the corresponding workspace and project files for Microsoft* Visual C++* 6.0. This application can process a signal by the only one operation (addition by a constant) but you may add other operations in much the same way. Besides of standard MFC classes the sample involves following classes:

CIppsSignal class
contains signal data and provides easy calls to Intel® IPP functions
CIppsSignalStore class
provides Open and Save File commands
CIppsSignalDC class
provides signal viewer
CIppsAddCDlg class
gets parameters for ippsAddC_8u_C3R function by dialog
CIppsAddC class
processes signal by ippsAddC_8u_C3R function.

Additional information on this software as well as other Intel software performance products is available at http://developer.intel.com/software/products/.

Intel IPP Functions Used in the Using the Intel® IPP Signal Processing Functions with the MFC

ippsVectorJaehne function is used for new signal initialization.

ippsAddC function is used for signal processing.

ippGetStatusString function is used for error diagnostics.

Installation

Extract all files in w_ipp-samples_*.zip to a desired folder. Please make sure to preserve the directory structure. The files for this sample will be found in the .\ipp-samples\signal-processing\signal-processing-mfc folder.

How to Build Source Code

Set up your build environment by creating an environment variable named IPPROOT that points to the root directory of your Intel® IPP installation. For example: IPPROOT=C:/Program Files/Intel/IPP/5.1/ia32/

Open workspace file ippsSample.dsw by Microsoft* Visual C++* 6.0 or Microsoft* Visual C++* .NET 2003. Build application as usual.

Running the Sample

In order to run this sample, the Intel® IPP DLLs must be on the system's path. DLLs are located in bin subdirectory of the root directory of the Intel® IPP installation (IPPROOT). This can be done by setting the PATH environment variable manually or invoking the batch file:

%IPPROOT%\tools\env\ippenv.bat

When built, windows application ippsSample.exe is appeared in .\ipp-samples\signal-processing\signal-processing-mfc\Release or .\ipp-samples\signal-processing\signal-processing-mfc\Debug directory. Run this application.

In the beginning of the session a new real Jaehne signal of the 16s type and length equal to 512 is created.

To create a new Jaehne signal, open a .wav file with a signal, or save signal to a .wav file using the standard Toolbar buttons or File Menu.

To view active signal properties (length, type and file path, if any), select "File\Properties" from the menu.

To process the signal by the ippsAddC_16s_Sfs function:

·         Activate the source signal

·         Select " Process\ippsAddC_16s_Sfs" from the menu. If this button appears in gray color, the active signal is not the ipp16s type

·       The dialog window with the function parameters will appear

·       Enter the value to add and the scale factor in the dialog window and click on the ‘OK’ button

·       The destination signal will be created
